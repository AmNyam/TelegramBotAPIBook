Telegram Bot API

Telegram Bot API - это интерфейс на базе HTTP, созданный для разработчиков, заинтересованных в разработке ботов для Telegram.

История изменений Bot API

Февраль 13, 2018

Bot API 3.6.

  • Добавлена поддержка форматирования текста в подписях к мультимедиа. Укажите не обязательный параметр parse_mode (Markdown или HTML), когда вы добавляете подпись.
  • В супергруппах, если бот получает сообщение, которое является ответом, он также получит сообщение, на которое отвечает это сообщение, - даже если исходное сообщение недоступно из-за настроек конфиденциальности бота. (Другими словами, ответ на любое сообщение в супергруппе сообщением, в котором упоминается бот или команда для бота действует как пересылка исходного сообщения боту).
  • Добавлено новое поле connected_website для объекта Message. Бот получит сообщение с этим полем в приватном чате, когда пользователь пройдет авторизацию на сайте с подключенным Login Widget и позволяет отправлять сообщения от вашего бота.
  • Добавлен новый параметр supports_streaming в метод sendVideo и поле с тем же именем для объекта inputMediaVideo.

Платформа для Telegram-ботов официально запущена.

Авторизация бота

При создании бота, ему присваивается уникальный token. Token выглядит примерно так 123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11

Отправка запросов

Все запросы к Telegram Bot API должны обслуживаться через HTTPS и должны быть представлены в следующей форме: https://api.telegram.org/bot<token>/METHOD_NAME. Пример:

https://api.telegram.org/bot123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11/getMe

Telegram Bot API поддерживает методы GET и POST HTTP и четыре способа передачи параметров в запросах Bot API:

  • Строка запроса URL
  • application/x-www-form-urlencoded
  • application/json (except for uploading files)
  • multipart/form-data (use to upload files)

Ответ содержит объект JSON, который всегда имеет логическое поле 'ok' и может иметь необязательное строковое поле 'description' с описанием результата. Если 'ok' равно true, запрос был успешным, и результат запроса можно найти в поле 'result'. В случае неудачного запроса 'ok' равно false, и ошибка описывается в поле 'description'. Также возвращается целочисленное значение 'error_code', но его содержимое может быть изменено в будущем. Некоторые ошибки могут также иметь необязательные поля 'parameters' типа ResponseParameters, которые могут помочь автоматически обработать ошибку.

  • Все методы в API Bot регистронезависимы.
  • Все запросы должны быть выполнены с использованием UTF-8.

Выполнение запросов при получении обновлений

Если вы используете webhooks, можно выполнить запрос к интерфейсу Bot API при отправке ответа на webhook. Положительным моментом является то, что нужно меньше запросов, недостатком - невозможно узнать, что такой запрос был успешным или получить его результат. Для передачи параметров можно использовать POST-запрос на https://api.telegram.org/bot<token>/method.

Также можно использовать запрос с наргрузкой application/json, application/x-www-form-urlencoded или multipart/form-data при этом метод, который нужно вызвать указывается в параметре запроса 'method'.

Получение обновлений

Существует два взаимоисключающих способа получения обновления списка сообщений для бота - метод getUpdates, и метод Webhooks. Входящие сообщения хранятся на сервере не дольше 24 часов, либо пока бот не получит их одним из способов. Оба метода в результате возвращают сериализованные JSON-объекты Update.

Объект Update

Этот object представляет обновление списка сообщений.
В любом представлении объекта Update может быть только один необязательный параметр.

Поле Тип Описание
update_id Integer Уникальный идентификатор обновления списка сообщений. Идентификаторы обновления начинаются с определенного положительного числа и увеличиваются последовательно. Этот идентификатор особенно удобен, если используется Webhooks, он позволяет игнорировать повторяющиеся обновления или восстанавливать их правильную последовательность.
message Message Необязательный параметр. Новое входящее сообщение любого типа — текст, фото, стикер и т.д.
edited_message Message Необязательный параметр. Отредактированная версия сообщения, которая известна боту
channel_post Message Необязательный параметр. Новый пост канала рассылки любого типа — текст, фото, стикер и т.д.
edited_channel_post Message Необязательный параметр. Отредактированная версия поста канала рассылки, которая известна боту
inline_query InlineQuery Необязательный параметр. Новый входящий встроенный запрос
chosen_inline_result ChosenInlineResult Необязательный параметр. Результат встроенного запроса, который был выбран пользователем и отправлен его чат-партнеру.
callback_query CallbackQuery Необязательный параметр. Новое входящее сообщение обратной связи
shipping_query ShippingQuery Необязательный параметр. Новый входящий запрос на отправку товара. Только для счетов-фактур с гибкой ценой
pre_checkout_query PreCheckoutQuery Необязательный параметр. Новый входящий запрос предварительной проверки заказа товара. Содержит полную информацию о проверке заказа товара

Метод getUpdates

Данный метод используется для получения входящих обновлений посредством технологии long polling. Метод возвращает массив объектов Update.

Параметр Тип Использование Описание
offset Integer Необязательный параметр Возвращает идентификатор первого в очереди объекта Update, значение данного идентификатора должно быть больше на один, чем значение предыдущих идентификаторов. По-умолчанию, возвращаются обновления, начинающиеся с самого раннего неподтвержденного обновления. Объект Update считается полученным, как только вызывается метод getUpdates с параметром offset, превышающим update_id. Отрицательное значение параметра offset может быть указано для получения массива обновлений равному смещению -offset от конца очереди обновлений. Все предыдущие обновления будут забыты.
limit Integer Необязательный параметр Ограничивает количество получаемых обновлений. Принимаются значения от 1 до 100. По-умолчанию, 100.
timeout Integer Необязательный параметр Таймаут в секундах для запросов long polling. По-умолчанию, значение равно 0 - короткий опрос. Значение должно быть положительным, короткий опрос должен использоваться только для тестирования.

Примечание
1. Этот метод не будет работать, если настроен исходящий webhook.
2. Чтобы избежать дублирования обновлений, пересчитайте offset после каждого ответа сервера.

Метод setWebhook

Данный метод отправляет входящие обновления по адресу, указанному в параметре url посредством HTTPS POST-запроса в формате сериализованного JSON объекта Update. В случае неудачного запроса попытки отправки запросов повторяются несколько раз. В случае успешного выполнения метод возвращает true.

Если вы хотите убедиться, что запрос Webhook поступает из Telegram, мы рекомендуем использовать секретный путь в URL-адресе, например. https://www.example.com/<token>. Поскольку никто не знает token вашего бота, вы можете быть уверены, что это мы.

Отправка файлов

Существует три способа отправки файлов (фотографии, наклейки, аудио, мультимедиа и т.д.):

  1. Если файл уже хранится где-то на серверах Telegram, его не нужно перезагружать: каждый файловый объект имеет поле file_id, просто передайте этот file_id как параметр вместо загрузки. Никаких ограничений на файлы, отправленные таким образом, нет.
  2. Укажите Telegram HTTP URL-адрес файла в сети. Telegram загрузит и отправит файл. Максимальный размер 5 МБ для фотографий и макс. 20 МБ для других типов контента.
  3. Загружайте файл с использованием multipart/form-data обычным способом загрузки файлов через браузер. Максимальный размер для фотографий - 10 МБ, для других - 50 МБ.

Загрузка с использованием file_id

  • Невозможно изменить тип файла при отправке с помощью file_id. То есть видео нельзя отправить в виде фотографии, фото не может быть отправлено в качестве документа и т.д.
  • Невозможно повторно отправить эскизы.
  • Повторная отправка фото с помощью file_id пошлет все его размеры.
  • file_id уникален для каждого отдельного бота и не может быть перенесен с одного бота на другой.

Отправка с помощью HTTP URL-адреса файла в сети

  • Целевой файл должен иметь правильный MIME-тип (audio/mpeg для sendAudio и т.д.).
  • Метод sendDocument отправляет по URL-адресу только файлы с расширениями gif, pdf и zip.
  • Чтобы использовать метод sendVoice, файл должен иметь тип audio/ogg и иметь размер не более 1 МБ. Голосовые заметки размером 1-20MB будут отправляться в виде файлов.

Настройки форматирования сообщений

Telegram Bot API поддерживает базовое форматирование сообщений. Вы можете использовать жирный и курсивный текст, а также встроенные ссылки и предварительно отформатированный код в сообщениях ваших ботов. Клиенты Telegram отобразят их соответствующим образом. Вы можете использовать форматирование в стиле Markdown или в стиле HTML.

Обратите внимание, что клиенты Telegram покажут предупреждение пользователю, прежде чем открывать встроенную ссылку («Открыть ссылку?» Вместе с полным URL-адресом).

Стиль Markdown

Чтобы использовать этот режим, установите параметр parse_mode метода sendMessage в значение Markdown. Используйте следующий синтаксис:

*bold text*
_italic text_
[text](http://www.example.com/)
`inline fixed-width code`
```text
pre-formatted fixed-width code block
```
Стиль HTML

Чтобы использовать этот режим, установите параметр parse_mode метода sendMessage в значение HTML. В настоящее время поддерживаются следующие теги:

<b>bold</b>, <strong>bold</strong>
<i>italic</i>, <em>italic</em>
<a href="http://www.example.com/">inline URL</a>
<code>inline fixed-width code</code>
<pre>pre-formatted fixed-width code block</pre>

Примечание:

  • Теги не должны быть вложенными.
  • Все символы, которые не являются частью тега или объекта HTML (<, >, &), должны быть заменены соответствующими объектами (< на &lt;, > на &gt;, & на &amp;).
  • Поддерживаются все числовые объекты HTML.
  • API в настоящее время поддерживает следующие именованные объекты HTML: &lt;, &gt;, &amp; и &quot;.

Игры

Ваш бот может предлагать пользователям играть в одиночные и многопользовательские HTML5-игры. Создавайте игры через @BotFather с помощью команды /newgame. Обратите внимание, это действие требует ответственности: нужно будет разработать правила для каждой игры, которые будут предлагать ваши боты.

  • Игры - это новый тип контента в Telegram, представленный объектами Game и InlineQueryResultGame.
  • Как только вы создали игру через @BotFather, вы можете отправлять игры в чаты в виде обычных сообщений с помощью метода sendGame или использовать встроенный режим с объектом InlineQueryResultGame.
  • Если вы отправляете игровое сообщение без каких-либо кнопок, у него автоматически будет кнопка 'Play GameName'. Когда эта кнопка нажата, ваш бот получает CallbackQuery с параметром game_short_name запрошенной игры. Вы указываете правильный URL для этого конкретного пользователя, и приложение открывает игру в браузере в приложении.
  • Вы можете вручную добавить несколько кнопок в свое игровое сообщение. Обратите внимание, что первая кнопка в первой строке всегда должна запускать игру, используя параметр callback_game в объекте InlineKeyboardButton. Вы можете добавить дополнительные кнопки в соответствии со вкусом: например, для описания правил или для открытия официального сообщества игры.
  • Чтобы сделать вашу игру более привлекательной, вы можете загрузить анимацию GIF через @BotFather, которая демонстрирует игру пользователям (пример, Lumberjack).
  • В игровом сообщении также будут отображаться лучшие результаты для текущего чата. Используйте setGameScore для отправки высоких результатов в чат с игрой, добавьте параметр edit_message, чтобы автоматически обновлять сообщение с текущей табло.
  • Используйте метод getGameHighScores для получения данных с высокими результатами в игре.
  • Вы также можете добавить дополнительную кнопку совместного доступа, чтобы пользователи могли делиться своим лучшим счетом с разными чатами.
  • Для примера, работы игровых ботов добавьте ботов @gamebot и @gamee.
